Kubernetes Probes Demo

Node.js Microservice for Testing Health Checks

About This Microservice

This Node.js microservice is designed to demonstrate Kubernetes health check mechanisms including startup probes, readiness probes, and liveness probes. It provides endpoints that simulate healthy and unhealthy states to help you understand how Kubernetes manages container lifecycle.

Startup Probe

Checks if the application has started successfully. Kubernetes waits for this probe to succeed before checking readiness and liveness probes. Useful for slow-starting containers.

Readiness Probe

Determines if the container is ready to accept traffic. If it fails, Kubernetes removes the pod from service endpoints, preventing traffic from reaching it until it recovers.

Liveness Probe

Checks if the container is still running properly. If it fails, Kubernetes restarts the container to recover from deadlocks or unresponsive states.

Available API Endpoints

GET /startup

Startup probe endpoint - Returns 200 when healthy, 503 after crash

GET /readiness

Readiness probe endpoint - Returns 200 when ready, 503 after crash

GET /liveness

Liveness probe endpoint - Returns 200 when alive, 503 after crash

GET /status

Returns detailed service status including uptime, CPU, and memory usage

GET /crash

⚠️ Triggers crash mode - All probe endpoints will start returning 503 errors

Response

How It Works

Normal Operation

All probe endpoints return HTTP 200 with a success response, indicating the service is healthy and ready to handle traffic.

Crash Triggered

When you call the /crash endpoint, the service enters "crash mode" and sets an internal flag.

Probe Failures

After crash mode is activated, all probe endpoints return HTTP 503 (Service Unavailable), simulating an unhealthy application state.

Kubernetes Response

Kubernetes detects the failing probes and takes action: removes pod from service (readiness) or restarts the container (liveness).

Learning Objectives

  • Understand how Kubernetes uses probes to manage container lifecycle
  • Observe the difference between startup, readiness, and liveness probes
  • Test how Kubernetes responds to failing health checks
  • Practice configuring probe parameters (timeout, period, threshold)
